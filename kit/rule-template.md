# 🎁 Rule テンプレート

特定のファイル・パスにだけ効かせたい制約を書くファイル。`paths` を指定すると、そのパスに触るときだけ読み込まれるので、無駄なコンテキストを使わない。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: `.claude/rules/<rule-name>.md` として保存。`paths` を書けば path-scoped（推奨）、書かなければ CLAUDE.md と同じく常時ロードになる。

```text
---
paths:
  - "src/api/**"
  - "**/*.handler.ts"
---

# API ハンドラの制約

- すべての API ハンドラは、処理の前に Zod で入力を検証すること
- DB マイグレーションは追記のみ（既存マイグレーションを書き換えない）
- 外部APIの呼び出しには必ずタイムアウトとリトライ上限を設定する
- 秘密情報（APIキー・トークン）をログやレスポンスに出さない
```

**ポイント**: パス限定ルールは「そのファイルを触るときだけ」読み込まれる。ドキュメントだけ編集するセッションでは API ルールが読み込まれず、コンテキストが軽くなる。「絶対に止めたい」ことはルール（指示）ではなく Hook（決定論的ガードレール）で止めるのが確実。
