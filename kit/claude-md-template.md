# 🎁 CLAUDE.md テンプレート

プロジェクトのルートに置く、セッション開始時に常時読み込まれる「事実」の置き場。ビルドコマンド・構成・規約など、Claudeが常に知っておくべき土台を書く。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: リポジトリのルートに `CLAUDE.md` として置く。200行以内を目安に。手順（デプロイ等）は書かず Skill へ、パス限定の制約は Rule へ分離する。

```text
# <プロジェクト名>

## 概要
- 何をするプロジェクトか（1〜2行）:
- 技術スタック:

## ディレクトリ構成
- src/ …
- tests/ …
- （monorepo ならワークスペース構成も）

## ビルド・実行コマンド
- インストール:
- 開発サーバ:
- テスト:
- Lint / Format:

## コーディング規約
- 命名 / フォーマット:
- コメントは「なぜ」を書く。「何を」は書かない
- エラーハンドリングはシステム境界（入力・外部API）にのみ

## チームの約束（norms）
- コミットメッセージの形式:
- レビューの流れ:

---
## ここに書かないもの（分離先）
- 手順・ワークフロー（デプロイ、リリース、レビュー）→ Skill（.claude/skills/）
- 特定パスだけの制約（例: src/api/** は Zod必須）→ path-scoped Rule（.claude/rules/）
- 「絶対にやってはいけない」ガードレール → Hook（PreToolUse）＋権限設定
- 個人的なトーン・好み → ユーザーレベルの個人設定
```

**ポイント**: CLAUDE.md は常時コンテキストを消費する。肥大化させず「事実」に絞り、手続きは Skill、禁止事項は Hook に逃がすのが公式の考え方。
