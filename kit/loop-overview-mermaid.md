# 🎁 Loop 全体構成図（Mermaid）

トリガー → 実行役 → 検証役 → （未達なら戻る／達成なら完了）→ 記録、という「良いループ」の骨格を1枚に描いた Mermaid 図。内側ループ（1タスク）と外側ループ（次のトリガーを待つ）の関係も表現。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: GitHub / Notion / Obsidian など Mermaid 対応の場所にそのまま貼ると図として描画される。

```text
flowchart TB
    subgraph OUTER[外側のループ: 次のトリガーを待ち続ける]
      T([トリガー<br/>人のプロンプト / 時間 / イベント]) --> D[実行役エージェント<br/>Doer]
      subgraph INNER[内側のループ: 1タスクを完了させる]
        D --> V{検証役<br/>Verifier<br/>完了条件を満たした?}
        V -->|未達| F[差分を修正] --> D
      end
      V -->|達成| L[(共有メモリ / LOG<br/>判断・変更を記録)]
      L --> R[人間がレビュー<br/>要所を確認]
      R --> T
    end
    G[[ガードレール: Hook + 権限<br/>絶対にやってはいけないことを止める]] -.- INNER
    ML[["まずシンプルに始める<br/>停止条件は必ず明確に"]] -.- OUTER
```

**ポイント**: 心臓部は「V →（未達）→ D」の戻り矢印。完了条件を満たすまで回るこの部分が、ループを“ただの1回実行”と分ける。検証役を実行役と分けると、自己採点バイアスが減って品質が安定する。
