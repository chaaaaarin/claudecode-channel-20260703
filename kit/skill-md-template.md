# 🎁 SKILL.md テンプレート

手順（ワークフロー）を再利用可能な形で書くファイル。呼び出し時にだけ読み込まれるので、コンテキストコストが低い。検証手順をここに書くと、Claude自身の自己検証が効くようになる。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: `.claude/skills/<skill-name>/SKILL.md` として保存。名前と説明で自動マッチ、またはスラッシュコマンドで起動する。

```text
---
name: verify-frontend-change
description: フロントの変更を検証する手順。UIを変えたら毎回この手順で自己検証する。
---

# フロントエンド変更の検証手順

この手順は、UIに変更を加えたあと「見た目・動作・エラー」を自分で確認するためのもの。
どれか1つでも失敗したら、直して最初からやり直す。中途半端なまま返さないこと。

## 手順
1. 開発サーバを起動し、対象ページを開く
2. 変更箇所を操作し、before/after のスクリーンショットを撮る
3. ブラウザのコンソールを確認する（エラー・警告ゼロが条件）
4. Chrome DevTools MCP で Core Web Vitals を計測する
5. 関連するテストを実行し、すべてパスすることを確認する

## 完了条件（定量的に）
- [ ] コンソールエラー 0件
- [ ] 対象テストがすべてパス
- [ ] レイアウト崩れなし（スクショで確認）

## 失敗したら
- 原因を特定して修正 → 手順1からやり直す
- 3回試して直らなければ、状況と試したことを添えて人間に戻す
```

**ポイント**: 検証は「定量的な基準」ほどClaudeが自己判定しやすい（エラー0件・スコア90以上など）。手順を SKILL.md に固定すると、毎回同じ品質でチェックが回る。
