# 🎁 Human-Agent Team 設計テンプレート

人間とAIエージェントが同じチームで働くための「名簿・北極星・信頼の広げ方」を決めるテンプレート。ループを個人の作業から、チームの仕組みへ育てるときに使う。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: チームのドキュメント（Slack canvas・Notion 等、エージェントが読める場所）に置く。「書かれていなければ存在しない」——エージェントは検索できるテキストからしか状況を理解できない。

```text
# Human-Agent Team 設計シート

## 1. 北極星（North Star）
- 事業・ミッションに根ざした長期ゴール（1文）:
- これを共有すると、優秀なエージェントは自分から次の作業を提案しはじめる

## 2. 名簿（Roster）— 誰が何を持つか
| メンバー | 人/AI | 担当タスク | 使えるツール |
|---|---|---|---|
| 例: レビュー担当 | AI | PRの初回レビュー | GitHub, テスト実行 |
|  |  |  |  |
- 人間もエージェントも同じスレッドで働き、誰でも途中から続けられるようにする

## 3. 公開性（Work in public）
- [ ] 意思決定・議事・ドキュメントを検索可能な場所に残す
- [ ] 共有はアイテム単位でなくワークスペース単位の境界で管理
- [ ] エージェントを主な読み手と想定してドキュメントを書く

## 4. 検証の仕組み（Verification）
- [ ] Doer-Verifier（起案役と検証役を分ける）
- [ ] 完了条件・チェックリスト・ルーブリックを用意
- [ ] 重要な判断はまとめて人間レビューに回す

## 5. 信頼を段階的に広げる（Trust over time）
- 最初: すべての出力を人間がレビューしフィードバック
- 次: 実績が出たタスク種別から自律範囲を広げる
- 定例: 週次で「教訓と失敗」を記録させ、同じ失敗を繰り返さない
```

**ポイント**: これは新しい話ではなく「明確な役割・良いドキュメント・共通の品質基準・失敗からの学習」という普遍のチーム論。エージェントが加わることで、それらが“あった方がいい”から“必須”に変わる。
